Stock Market

Stock Market Simulation Program

View Full Source on GitHub

One of the greatest full programs I've built to date. This program was inspired by Yahoo! Finance, which offers a great (and free) API for obtaining real-time information about the stock market. The program is a stock market simulation, allowing users to lookup real information about existing stocks, and then "buy" and "sell" them (again, simulated) to try and make a maximum profit. This program pushed me to master thread-safe class design, proper programming structure, and database access, just to name a few. This program has been the backbone of my growth as a programmer, as I initially completed a much simpler version of this months ago, yet kept coming back and using it as the basis for growing my skills and improving as a developer. As Version 2.0, a complete re-write from the ground up, approaches completion (as of 8/30/2015 it's a few weeks away from completion), it has become a showcase of the many skills I have gained from my experience.

Key Components

Safely Concurrent

User actions are quickly offloaded from the EventDispatchThread to background threads for processing, with all gui changes being pushed back onto the EDT before execution. All property models and shared objects are properly synchronized to protect their state, with synchronized blocks covering as little as possible to maximize performance. Immutable objects and local variables are used as often as possible to further minimize the need for synchronization by reducing mutable state.

Stores Data in a MySQL Database

Using the JDBC, all data for this program is stored in a MySQL database. Database property information is loaded from a separate file at runtime, and periodic connections are acquired to load, save, and modify information. All database operations are wrapped in a Data Access Object, thus encapsulating all code for querying the database within the DAO and providing simple methods for the controller to invoke. Finally, the database connection itself has its accessed controlled by a Semaphore with a set number of permits.

Beautiful UI With Compartmentalized Design

Built using Swing and the Nimbus LookAndFeel, the GUI classes use a wide range of components and icons to achieve an elegant appearance. Structurally, aggregation is the design style used to maintain maximum compartimentalization of the code. All classes that depend on a component from another class are coded with a blank component in that place, and are capable of being initialized with or without those other classes. Setter methods are provided to inject the component into the UI and trigger the re-assembly of the Swing object to incorporate the necessary item at runtime. This allows for the various UI classes to be easily maintained separately, or even swapped out at will.

Uniform Dialog API

In many cases I needed a more complex dialog than what was offered by JOptionPane. To avoid inconsitency in appearance, a dialog API was designed to ensure that subclasses could produce dialogs as simple or complex as needed by implementing a handful of methods. Most importantly, no matter how much or how little the dialog contained, the basic visual style would always be consistent.

Retrieves Data From the Web

Establishes a connection with Yahoo! Finance's servers to retrieve the data necessary to run the program. Information is downloaded as a .csv, parsed, and returned to the caller so it can be properly stored. The downloader is designed to not only encapsulate all details of the web connection, but also operate with all details of the web query proivided entirely by the Stock object. The downloader merely parses the query to ensure its compatible with the web service, downloads the data, and returns what it receives to the caller.

Multi-Lingual Support

While additional languages have yet to be included, this program 100% supports multi-lingual implementation. No text is hard-coded into the GUI, and instead is stored in properties files and accessed using the ResourceBundle class. A special shared Language module wraps around the bundle so that not only can the single bundle be accessed by multiple classes, but the language setting can be changed during runtime and the GUI re-loaded with a new text.

Utilizes my MVP Framework

My Model-View-Presenter framework was designed while working on this project. Originally just an effort to enforce MVP design throughout the program, this framework was done abstractly enough that I chose to extract it from this program and make it a stand-alone project for future re-usability. It's full described in its own page on this site, but it is an expansion on the basic MVP pattern, using reflection and Java's listener system to significantly reduce dependencies between models, views, and controllers compared to the standard MVP design.
