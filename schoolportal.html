School Portal

School Portal Web Application

View Full Source on GitHub

NOTE: Currently a work-in-progress, and only deployed on local development server.

A full Java web application, built around the Spring MVC framework. This demo application is a web portal for a fictional high school, providing tools for both administrators and students. Administrators can register and modify available courses and override student selections on course schedules, while students can simply choose their schedules from the available courses.

It makes full use of the power of the Spring framework. The IOC Container, ORM tools, Transaction management, testing, web and web-MVC tools. It is unit tested through JUnit. On the backend it is supported by a MySQL database, and ORM mapping is handled by the Hibernate framework using JPA annotations. JSP pages are used as the primary view technology, with full use of JSTL and Spring tag libraries. All pages are internationalized, and all controllers are designed with full thread-safety in mind.

Key Components

Spring-Driven Java Web Application

Spring is at the heart of this application. The deployment descriptor is set to pass all activity to Spring's DispatcherServlet, which in turn uses mapped Spring controllers to perform actions. XML configuration is used for Spring API beans, while application beans are annotation-driven.

Service Layer API

The service layer is also extremely flexbile, built on generic interfaces that are designed to handle a wide range of entities and their DAOs. These generic methods accept type arguments to determine which entity is being persisted (and which DAO is needed for the operation). If additional entities/DAOs are added in the future, the only changes that need to be made are to the implementation, no API methods will need to be added/changed.

JSP Pages for Views

Java Server Pages are used for the view technology for this application. The JSTL and Spring tag libraries are used in addition to standard HTML tags to build the UI, coupled with CSS stylesheets. The look-and-feel is a combination of homemade UI components and styles and Bootstrap elements. The pages dynamically update based on data passed from the application, and all UI text is internationalized.
