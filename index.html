Stock Market

Yahoo! Finance offers a great (and free) API for obtaining real-time information about the stock market. This program is a stock market simulation, allowing users to lookup real information about existing stocks, and then "buy" and "sell" them (again, simulated) to try and make a maximum profit. This was one of the early programs I created, and was completed months ago. However, it is currently in the midst of a re-write to implement a wide range of new techniques that I have learned since its original creation. The Version 2.0 re-write is roughly 80% complete as of 8/27/2015, and should be finished within the next several weeks.

Key Components:

  • Utilizes my MVP Framework. That project is detailed in full below.
  • Beautiful UI utilizing the full power of Swing. I'm particularly proud of the uniform dialog API I created, which provides a framework for a uniform style of dialog.
  • Multi-threaded and thread-safe, allowing for concurrent actions to be performed as needed.
  • Uses HTML connections to pull data from Yahoo! Finance and display it in the program.
  • Uses the JFreeChart library to display a long-term history of the stock price.
  • Saves and loads the state of each stock portfolio in a MySQL database using the JDBC.
  • Utilizes industry standard design patterns, including: Singleton, Factory, Data Access Object, Decorator, Monitor, and MVP.
  • Multi-lingual support through Locale-based property files and ResourceBundles.

MVP Framework

When I began studying industry standard Java design patterns, Model-View-Presenter was one that I truly fell in love with. Nevertheless, I felt the basic design that's shown all over the web didn't go far enough in separating its components and reducing the coupling between them, especially with the views and controllers. This simple framework has the traditional MVP design at its core, and then builds on it using Java's event listener system, advanced reflection techniques, and safe concurrency to provide a far more advanced framework for implementing the MVP pattern, which is capable of being used by any program.

Key Components:

  • At its core, always adheres to the Model-View-Presenter design pattern, with the controller handling all communication between the models and views.
  • Utilizes the ActionListener interface to more fully separate the controller from the view, allowing for the controller to receive all events from view classes without needing any specific knowledge of their implementation.
  • Provides implementations for view GUI classes and for dialogs as well.
  • Supports concurrency through the AbstractConcurrentListenerController class, which handles the process of moving tasks from the EventDispatchThread to background threads, and then back to the EDT afterwards. The implementing class need only concern itself with writing code for those background processes.

Money and Currency Conversion

What started as my first experiment with accessing a web based service and retrieving information transformed into a pretty awesome little library. It ties BigDecimal values to currency types, which enables a number of special operations. Essentially it allows amounts of money of various currency types to be compared, combined, and otherwise operated on by a program.

Key Components:

  • Money objects must always have a BigDecimal value and a CurrentyType. Any change to the type results in a conversion operation on the value.
  • Objects of different currency types can be used together in mathematical operations (add/subtract/multiply/divide), which will automatically perform the necessary conversions to produce the correct result.
  • Amounts of money with different currency types can be compared to one another, using the Comparable interface and propper conversions.
  • Currency conversion rates are downloaded when needed from Yahoo! Finance.
  • Currently supports 24 different currency types, but could easily support all currency types, so long as the enum constant values conform to the ISO4217 standard.

TS-Inverted Apps

My oldest project, doesn't use Java or SQL but "Edify", a special scripting language used for Android installer scripts. This script is designed to install a series of themed versions of popular Android apps, and was developed in cooperation with the theme developers. It utilizes the Android Aroma Installer (from developer Amarulz) to provide the user with a simple interface to select which apps to install, defining a series of properties files along the way. It then executes the installation process based on the items the properties files indicate were selected. Unfortuantely, I had to abandon this project when my work at the time became too busy for me to continue it.

Key Components:

  • Option to do a full GApps installation along with installing the specified apps.
  • Full slate of TeamBlackout themed apps.
  • Aroma Installer gives the user the ability to pick and choose which apps to install.
